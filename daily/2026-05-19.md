### 阅读：Prompt

读了 [AI × Web3 School Handbook — Prompt 章节](https://aiweb3.school/zh/handbook/ai/prompt/)

### 关键概念 / 收获

1. **Prompt 的本质逻辑** — 不是"命令"，而是"给模型搭上下文框架"。模型不是执行指令，而是在给定上下文中做概率续写。好的 prompt 是在帮模型缩小可能性空间。
2. **结构化 Prompt** — 角色设定 + 任务描述 + 输出格式 + 约束条件，这套框架在不同模型上通用度很高，但各模型对格式敏感度不同。
3. **Chain-of-Thought（思维链）** — 让模型"一步步思考"能显著提升复杂推理任务的准确率，尤其在 DeepSeek 和 ChatGPT 上效果明显。
4. **Few-shot vs Zero-shot** — 给例子（few-shot）在格式控制任务上比纯描述 zero-shot 更稳定，但会多耗 token。需要根据任务复杂度权衡。
5. **鲁棒性问题是真问题** — 同一 prompt 在不同模型、甚至同一模型不同版本上的输出可能差异很大。好的 prompt 应该对细微措辞变化不敏感。

### 多模型 Prompt 实操对比

今天分别在 5 个模型上测试了同样的 prompt 任务（结构化输出 + 逻辑推理），记录如下：

| 模型 | 格式遵循度 | 推理能力 | 备注 |
|------|-----------|---------|------|
| **ChatGPT (GPT-4o)** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 最稳，结构化输出几乎零误差 |
| **DeepSeek** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 推理很强，但格式偶有偏差 |
| **Kimi** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 长上下文优势明显 |
| **豆包** | ⭐⭐⭐ | ⭐⭐⭐ | 日常问答够用，复杂推理略弱 |
| **Gemini** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 和 ChatGPT 接近，偶尔格式飘 |

**结论：** 没有"最好的模型"，只有"最合适的 prompt 策略"。在不同场景下需要针对性调参。
